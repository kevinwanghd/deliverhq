## 需求描述

### 背景
TodoApp 目前只有前端静态页面，需要实现后端 API 支持待办事项的持久化存储和管理。

### 功能要求
1. 创建待办事项（标题、描述、截止日期）
2. 查询待办事项列表（支持分页、按状态筛选）
3. 更新待办事项（修改内容、标记完成）
4. 删除待办事项（软删除）

### 非功能要求
- 接口响应时间 P95 < 100ms
- 支持并发请求 > 100 QPS
- 数据不丢失（PostgreSQL 持久化）

### 验收标准
- 所有 CRUD 接口通过集成测试
- 接口文档自动生成（OpenAPI）
- 错误响应遵循统一格式

- 修改：`src/main.py`（注册路由）
