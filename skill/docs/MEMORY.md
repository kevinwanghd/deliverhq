## 使用说明

### 何时更新此文件

- **开始新 CR**：更新"当前活跃 CR"
- **Gate 通过**：更新"最后通过的 Gate"
- **遇到阻塞**：添加到"当前阻塞"和"待确认问题"
- **做出决策**：添加到"最近决策"（同时更新 decisions.md）
- **Gate 失败**：自动追加到"最近失败"（通过 update_mistake_book.py）
- **识别风险**：添加到"当前风险"
- **CR 归档**：清空"当前活跃 CR"

### 读取顺序

Agent 应按以下顺序读取：

1. `AGENTS.md` — 行为规则和职责边界
2. `dir-graph.yaml` — 权限和路径定义
3. `docs/CONTEXT.md` — 项目背景（静态）
4. **`docs/MEMORY.md`** — 当前状态（动态）
5. 当前 CR 的具体文档

### 与其他文档的区别

| 文档 | 类型 | 内容 | 更新频率 |
|------|------|------|----------|
| `CONTEXT.md` | 静态知识 | 项目背景、技术栈、架构 | 很少（架构变更时） |
| `rules.md` | 静态规则 | 编码规范、约定 | 偶尔（规则演进） |
| `decisions.md` | 历史记录 | 所有架构决策 | 每个决策后追加 |
| `mistake-book.md` | 历史教训 | 错误案例库 | Gate 失败时自动追加 |
| **`MEMORY.md`** | **动态状态** | **当前进行中的 CR、问题、风险** | **每次状态变化** |
